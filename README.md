# BipolarMood (БАРсик)

Нативное Android-приложение для людей с биполярным аффективным расстройством (БАР) и их близких. BipolarMood сочетает КПТ-дневник, трекер настроения, фиксацию импульсивных желаний, напоминания о терапии, поддержку доверенных лиц и визуализацию долгосрочных трендов.

> Приложение не заменяет врача, психотерапию или экстренную помощь. Его цель - помочь пользователю замечать изменения состояния, фиксировать важные события и вовремя обращаться за поддержкой.

## Основные цели

- Быстро фиксировать настроение, симптомы и заметки в течение дня.
- Отслеживать импульсивные желания и оценивать их потенциальную проблемность.
- Напоминать о приеме лекарств и эскалировать пропуски доверенному лицу.
- Вести ежедневник с заметками и фотографиями.
- Показывать графики, тепловые карты и отчеты для врача.
- Дать близким read-only доступ к выбранным данным пользователя.
- Предоставлять быстрый Safety Plan при резком ухудшении или подъеме состояния.

## Технический стек

| Область | Технологии |
| --- | --- |
| Язык | Kotlin |
| UI | Jetpack Compose, Material 3, кастомная тема |
| Архитектура | MVVM + Clean Architecture |
| Локальное хранение | Room / SQLite |
| Dependency Injection | Hilt или Koin |
| Фоновые задачи | WorkManager |
| Push-уведомления | Firebase Cloud Messaging |
| Шаринг с близкими | Firebase Firestore или Supabase с read-only правилами |
| Фото | MediaStore, CameraX |
| Экспорт | PDF, CSV |
| Интеграции | Google Fit для сна, Google Drive Backup API опционально |

## Архитектура

Рекомендуемая структура модулей:

```text
app/
  presentation/      # Compose screens, navigation, ViewModel
  domain/            # Entities, Use Cases, business rules
  data/              # Room, repositories, remote sync, mappers
  notifications/     # FCM, WorkManager, medication reminders
  media/             # CameraX, MediaStore, image persistence
  export/            # PDF/CSV generation
  designsystem/      # Theme, colors, typography, reusable components
```

Ключевые принципы:

- Все персональные данные по умолчанию хранятся локально в Room.
- Синхронизация наружу включается только для сценариев read-only доступа близких.
- Use Case слой содержит правила оценки импульсов, эскалации лекарств и триггеры Safety Plan.
- UI не должен напрямую обращаться к Room, Firebase или WorkManager.
- Экспорт и шаринг должны явно показывать пользователю, какие данные будут переданы.

## Функциональные модули

### 1. Главный экран

- Крупный маскот БАРсик - снежный барс, отражающий последнее состояние.
- Быстрая кнопка `Зафиксировать состояние`.
- Красное сердечко для перехода в Safety Plan.
- Последние 3 записи настроения.
- Мини-график настроения за текущий день.
- Подсказки при резком подъеме, снижении или нарушении сна.

### 2. Трекер настроения

Диалог фиксации состояния включает:

- Точное время записи, по умолчанию текущее, с возможностью выбрать прошедшее время.
- Шкалу настроения от `-5` до `+5` с шагом `0.5`.
- Категорию состояния:
  - депрессия;
  - гипомания;
  - мания;
  - смешанное;
  - нейтральное.
- Чекбоксы симптомов:
  - нарушение сна;
  - повышенная говорливость;
  - раздражительность;
  - чувство вины;
  - дереализация / деперсонализация;
  - ускорение мыслей;
  - снижение потребности во сне.
- Свободное текстовое поле для заметок.

При оценке `<= -4` или `>= +4` приложение предлагает открыть Safety Plan.

### 3. Импульсы

Раздел для фиксации желаний и действий, которые могут быть связаны с гипоманией, манией, смешанными состояниями или стрессом.

Поля записи:

- Что сделал или захотел пользователь.
- Стоимость, опционально.
- Категория:
  - хобби / мелочи;
  - одежда / косметика;
  - техника / гаджеты;
  - путешествия / билеты;
  - кредиты / финансовые обязательства;
  - отношения / крупные решения;
  - другое.
- Дата и время.
- Автоматическая предварительная оценка проблемности от `1` до `10`.
- Ручная корректировка автора и комментарий.
- Отдельная оценка близкого.

Автоматическая оценка учитывает:

- стоимость: `0-6` баллов;
- категорию: `0-6` баллов;
- ключевые слова вроде `все`, `сейчас`, `немедленно`, `навсегда`: до `+3`;
- текущее настроение: если за последние 2 часа выраженный подъем, до `+2`.

Если пользователь сохраняет импульс при мании `>= +4`, приложение показывает предупреждение перед сохранением. Кнопка `Спросить близкого` отправляет доверенному лицу push-уведомление с просьбой оценить действие.

### 4. Лекарства

Функции:

- Добавление препарата:
  - название;
  - дозировка;
  - время приема;
  - частота: ежедневно, через день, по расписанию;
  - часовой пояс.
- Push-уведомление с действием `Принял(а)`.
- Повторное напоминание каждые 30 минут, если прием не отмечен.
- Эскалация после 5 пропущенных уведомлений:

```text
[Имя пользователя] не принял(а) [препарат] уже 2.5 часа. Пожалуйста, напомни.
```

- Календарь приемов с отметками.
- Статистика пропусков и приверженности терапии.

### 5. Read-only доступ для близких

Пользователь может сгенерировать уникальный токен или ссылку для доверенного лица.

Близкий после регистрации видит:

- график настроения;
- заметки пользователя только для чтения;
- импульсы с оценкой автора;
- возможность добавить свою отдельную оценку импульса;
- статус приема лекарств.

Пользователь может отозвать доступ в любой момент.

### 6. Дневник

Вкладка `Дневник` оформляется как вертикальная лента из вырванных тетрадных листов.

Каждая запись содержит:

- дату и время рукописным шрифтом;
- текстовую заметку;
- одно или несколько фото из галереи или камеры;
- возможность редактирования и удаления.

Фото отображаются как полароидные снимки:

- белая рамка;
- тень;
- легкий случайный поворот до `5°`;
- полноэкранный просмотр с зумом по нажатию.

Сценарии использования: питание, важные дела, фото принятой таблетки, заметки для КПТ, фиксация событий дня.

### 7. Аналитика и визуализация

Основные графики:

- Линейный график настроения по шкале `-5..+5`.
- Периоды: день, неделя, месяц, год.
- Маркеры приемов лекарств, импульсов и нарушений сна.
- Точечный график проблемности импульсов.
- Столбчатая диаграмма приверженности терапии.
- Совмещенный график настроения и сна.

Тепловая карта:

- Переключатели: час, день, неделя, месяц, полгода, год.
- Каждый квадрат представляет период:
  - час для 24 часов;
  - день для недели или месяца;
  - неделя для полугода;
  - месяц для года.
- Цвет от темно-красного `-5` через серый `0` до ярко-желтого `+5`.
- Пустые периоды отображаются черным.
- По нажатию показывается подсказка:
  - дата;
  - среднее значение;
  - количество записей;
  - связанные заметки.

Экспорт:

- PDF-отчет для врача.
- CSV с сырыми данными и значениями тепловой карты.

### 8. Safety Plan

Быстрый доступ с главного экрана по кнопке `Помощь` или красному сердечку.

Содержимое:

- контакты врача;
- контакты близких;
- кризисные номера;
- индивидуальные стратегии совладания;
- подсказки при резком ухудшении или подъеме.

Примеры проактивных подсказок:

- При резком подъеме: `Помни о последствиях импульсивных решений`.
- При снижении: `Ты не один, обратись к близким`.
- При пропуске сна: `Нарушение сна - частый триггер, постарайся отдохнуть`.

### 9. Сон и смешанные состояния

Функции:

- Ввод времени засыпания и пробуждения.
- Опциональная интеграция с Google Fit.
- Отметка маркеров смешанных состояний:
  - раздражительность + грусть;
  - эйфория + заторможенность;
  - беспокойство + подавленность.

### 10. Настройки и профили

Профиль пользователя:

- имя;
- аватарка из галереи или готовых иконок;
- дата рождения, опционально.

Профиль близкого:

- имя;
- аватарка;
- контактный телефон.

Настройки приложения:

- звуковые уведомления;
- тема: темная по умолчанию или светлая;
- часовой пояс;
- интервал напоминаний, по умолчанию 30 минут;
- экспорт и импорт данных;
- управление доступом близких.

## Дизайн-концепция

### Маскот и логотип

БАРсик - милый мультяшный снежный барс в минималистичном плоском векторном стиле с толстыми контурами.

Хвост маскота отражает настроение:

- нейтральное: опущен или слегка изогнут;
- депрессия: сильно опущен, грустные глаза;
- мания: поднят вверх, легкая улыбка.

На главном экране хвост анимируется через Spring-анимацию и реагирует на последнее зафиксированное состояние.

### Палитра

| Элемент | Цвет | HEX | Применение |
| --- | --- | --- | --- |
| Фон | Глубокий черный | `#0B0B0B` / `#121212` | Все экраны по умолчанию |
| Основной текст | Белый | `#FFFFFF` | Заголовки и основной текст |
| Второстепенный текст | Светло-серый | `#B0B0B0` | Подписи и даты |
| Границы | Темно-серый | `#2A2A2A` | Карточки, линии, разделители |
| Позитивный акцент | Ярко-желтый | `#FFD700` | Кнопки, достижения, высокий подъем |
| Риск / важное | Красный | `#FF3B30` | Пропуски лекарств, высокий риск, Safety Plan |
| Нейтральный акцент | Бирюзовый | `#4DD0E1` | Промежуточные состояния, графики |

Акценты должны занимать не более 10% экрана, чтобы не создавать лишнюю стимуляцию.

### Дофаминовые акценты

- Красное сердечко в правом верхнем углу открывает Safety Plan.
- При исправном приеме лекарств сердечко мягко пульсирует как позитивное подкрепление.
- Желтая звездочка появляется за регулярные записи, например за 7 дней подряд.
- В настройках можно кастомизировать БАРсика: добавить галстук-бабочку или шляпу.

### Типографика и иконки

- Основной шрифт: Inter или Roboto.
- Рукописный шрифт для дат дневника: Caveat или аналог.
- Иконки: Material Symbols в линейном стиле или кастомные иконки в стиле барса.

### Анимации

- Хвост БАРсика: Spring-анимация.
- Кнопка фиксации: легкая вибрация и короткая визуальная вспышка без звука.
- Переходы между экранами: бесшовные, без агрессивных слайдов.
- Новая запись в дневнике: плавное появление листа.

## Навигация

Нижняя навигация содержит 7 разделов:

1. `Главный` - быстрая фиксация, БАРсик, последние записи, мини-график.
2. `Календарь` / `Лента` - история настроения и фильтры.
3. `Графики` - линейные графики и тепловая карта.
4. `Дневник` - тетрадные листы, фото и заметки.
5. `Лекарства` - препараты, календарь приемов и напоминания.
6. `Импульсы` - желания, действия и оценки проблемности.
7. `Настройки` - профили, доступ близких, тема, звук, часовой пояс, экспорт.

## Модель данных

Базовые сущности:

- `MoodEntry` - настроение, категория, симптомы, заметка, дата и время.
- `ImpulseEntry` - описание, стоимость, категория, автоматическая и ручная оценки, оценка близкого.
- `Medication` - препарат, дозировка, расписание, часовой пояс.
- `MedicationIntake` - факт приема, пропуск, статус эскалации.
- `DiaryEntry` - текст, дата, список фото.
- `SleepEntry` - время сна и пробуждения, качество сна.
- `TrustedPerson` - данные близкого, уровень доступа, статус токена.
- `SafetyPlan` - контакты, кризисные номера, стратегии совладания.
- `UserProfile` - имя, аватарка, настройки.

## Приватность и безопасность

- Локальные данные должны храниться в Room.
- Синхронизация с Firebase / Supabase включается только после явного действия пользователя.
- Доступ близких ограничивается read-only данными и отдельной оценкой импульсов.
- Токены доступа должны быть отзывными.
- Экспорт PDF/CSV должен требовать подтверждения пользователя.
- Медицинские уведомления не должны раскрывать лишнюю информацию на заблокированном экране, если пользователь отключил подробный текст.

## Уведомления и фоновые задачи

- FCM используется для push-уведомлений между пользователем и близкими.
- WorkManager отвечает за повторные локальные напоминания о приеме лекарств.
- Повтор по умолчанию: каждые 30 минут.
- Эскалация: после 5 пропущенных напоминаний.
- Пользователь может отключить звук, изменить интервал и настроить часовой пояс.

## Статус репозитория

Репозиторий содержит рабочее Android-приложение на Kotlin + Jetpack Compose. Debug APK собирается локально и может быть установлен на Android-устройство.

Реализовано в текущем MVP:

- семь вкладок нижней навигации;
- локальная база Room для настроения, импульсов, лекарств, дневника, сна, профиля и доступа близких;
- быстрый диалог фиксации настроения;
- лента настроения и записи сна;
- график настроения, тепловая карта, график импульсов и приверженности терапии;
- дневник с текстом и фото из галереи в стиле полароидов;
- лекарства с отметкой приема, пропуском, счетчиком эскалации и WorkManager-уведомлением;
- импульсы с автоматической и ручной оценкой, а также отдельной оценкой близкого;
- настройки профиля, Safety Plan, токены read-only доступа близких и CSV-экспорт.

Интеграции Firebase / Supabase / FCM подготовлены архитектурно как следующий слой, но для немедленной установки и приватности MVP работает полностью локально.

## Сборка и установка

Подготовить Android SDK в workspace:

```bash
./scripts/setup-android-sdk.sh
```

Собрать debug APK:

```bash
./gradlew assembleDebug
```

Готовый APK:

```text
app/build/outputs/apk/debug/app-debug.apk
```

Установка на подключенное устройство:

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

Проверки, выполненные для текущей версии:

```bash
./gradlew assembleDebug
./gradlew lintDebug
```
